On this page
As soon a user login onto the Test Framework, there would be a list of projects showing up listing all projects that is being managed by your Test Framework.
Project's information
On the project's information board, you may find all information related to a project listed as below:
①，Name of the project, and by clicking at the link, a user could enter this project's test centre.
②，Project's description, you could refer to the description of a project here.
Please kindly be aware that if a project is configured with a github link, the description shall be pulled from the github repository directly.
③，Delete the Project, an administrator of Test Framework could delete a listed project with this button.
Add, Delete and Modify
A user could add, delete and modify a project on this dashboard.
Add a project
An administrator of Test Framework could add a new project with the ADD button in the right bottom corner show as below:
Test Framework accepts a project from a storage that is either a local hardware or a remote repository on github. Regretfully Test Framework do not accept projects stored with other platforms. However, there are possibilities that those are to be added in future.
And please kindly provide below information when a request needs to be added by CIDANA:
Storage's type Example:github or local hardware
codebase's locationexample: https://github.com/Cidana-Developers/CidanaTestPlatformRelease.git
Delete a project
As shown below, an administrator could delete an added project from the project's listed with a simple click:
please kindly be aware that a deleted project could not be restored again, as well as all testing results related to this project.
Modify a project
When a project is added, the information related to this project could be modified when required.
For those codebase that is deployed in a github repository, as long as the description and name is updated in the github, the inforamtion listed in project's list could be updated in a few hours.
As for those who has used the codebase that is deployed locally, they could simply modify the configuration file that is sent by CIDADA to save in the target project's root directoy.
In the case any further modification is required, please kindly contact your person of contact in CIDANA.
An administrator's account is needed to add, delete or modify a project, as in the situation that you requested the deployment service from CIDANA, please kindly contact your person of contact in CIDANA to complete the deployment of new project:
