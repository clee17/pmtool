Pre-requisites
Before deploy, please kindly make sure the target device meets the basic requirements:
OS
The Test framework runs on linux , including but not limited to Ubuntu and CentOS. It could be deployed to both a physical linux machine and a virtual one.
  • Ubuntu 16.04 or above (Recommend)
  • CentOS 7 or above
Git
The test framework could either be installed directly with a downloaded package or fetched from the github at the address of Cidana Test Frame. In the case a git is used, the version has to be higher than 2.7.
  • Ubuntu 16.04 or above (Recommend)
  • CentOS 7 or above
The followging command can check if the git is ready in your test machine.
git --version
If git is ready, you will get exact version number, like git version 2.7.4,else command not found will occurs.
please install git by command if necessary:
sudo apt install
Docker
If the package is to be installed in a docker, the docker has to meet below standards.
Docker 19.0 or above.
Please use the following command to check if the test machine have docker installed.
docker --version
If error command not found occurs, it means no docker environment is installed
If a docker needs to be installed , the following commands shall be required.
Install docker in ubuntu:
sudo apt install docker.io
Install docker in CentOS:
please refer to https://docs.docker.com/engine/install/centos/#install-using-the-repository to install docker in CentOS.
Get installation package
1. target directory
The test framework could be installed to any directory, however, the /home dir is highly as the target deployment directory.
By default, the /home dir will sohw up immediately after you login to your linux system. The directory could be reached by cd or cd ~ if otherwise.
2. Get the installation package from github
The installation package could be retrieved from github at below address:
git clone https://github.com/Cidana-Developers/CidanaTestPlatformRelease.git
3. Get the installation package from CIDANA
If you've purchased the frame from CIDNA, please kindly contact your person of contact in CIDANA to get the latest package.

Configuration (Optional)
Before deployment, we need to generate a config first. A utility is highly recommended which you could trigger as below:
cd ./CidanaTestPlatformRelease/scripts && sudo chmod +x *.sh && sudo ./config.sh -c {xyz}
Please note that:
  • Where {xyz} is a placeholder, please replace it with a real name for the config file.
  • config is generated when the Test framework gets deployed for the first time, it could be used in the next few deployement should it is deployed on the same machine.
The correct IP/domain & port of target machine needs to be set up according to instructions given by the config generation procedure.
After the config is generated, a new command of deploy CLI will show up. Please copy the command directly and boot the test frame.
Deployment
When there is a final command showing up in the configuration step, the deployment process shall be completed automatically.
Please note that:
  • option -uin deploy CLI tail will force download the latest docker's image when deploy, otherwise a local docker's image is preferred. Please download a docker image only when it does not exist in local.
Once the deployment is finished successfully, the entrance link of Test Framework shall be displayed on screen, just as below:
http://192.168.0.88:8083
Please kindly note that the address varies according to your configuration so please kindly copy the one on your screen instead of the one shown above, and open that link in a browser.
